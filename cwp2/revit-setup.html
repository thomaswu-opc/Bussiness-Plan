Part 2 · Revit Setup

CAD-Revit 映射的 Revit 设置步骤

这份说明把映射表里的 CAD 图层、线型、线宽、填充、标注信息，落到 Revit 的导入线宽、对象样式、VV/VG、导入类别与填充策略中，方便驻场支持时按步骤配置和复核。

01 · Before Revit

进入 Revit 前先锁定映射表

本地模板中的“总映射表”是主控表，“CAD资源字典、Revit样式字典、标注注释字典”是标准名称来源。建议先完成字段闭环，再打开 Revit 配置。

CAD

CAD资源字典

登记 CAD 图层、图层用途、线型、线宽、颜色/色号、打印样式、块/样式名和适用图纸。

Revit

Revit样式字典

登记对象样式、子类别、线样式、线宽、材质/填充、视图样板和导入的图元样式。

Anno

标注注释字典

统一标注类型、注释类型、字体、文字高度、颜色、标注线、标头/箭头和引线端点。

Map

总映射表

逐行确认映射编号、应用场景、CAD表达、Revit显示控制、映射类型、优先级、状态与来源依据。

映射表字段 进入 Revit 后对应设置 核对重点
CAD图层 / 线型 / 线宽 / 颜色 导入线宽、对象样式、导入的图元样式、VV/VG 导入类别 线宽号、打印粗细、颜色是否被视图覆盖
填充 / 图案 Revit 填充图案、填充区域、导入图元显示 是否需要跟随视口比例联动
标注类型 / 字体 / 文字高度 / 标头 标注样式、文字样式、箭头样式、视图比例 出图比例下是否与 CAD 标准一致
映射类型 / 优先级 / 状态 配置优先顺序与待确认清单 高优先级先配置，待定项不得进入模板固化

02 · Setup Steps

Revit 内部设置步骤

以下步骤按“项目通用设置 → 单次导入设置 → 视图显示控制 → 二维资源沉淀”组织，适合用于幕墙二维表达标准化试点。

STEP 1

配置导入线宽：Insert → Import panel → Import Line Weights

根据 Autodesk 官方说明，DWG/DXF 导入时每个图层会依据 CAD 的 pen number-line weight 设置映射到 Revit 线宽。先按映射表把 CAD 线宽归并到 Revit 线宽号，再保存为项目可复用的导入线宽配置。

  • 从映射表提取 CAD 线宽、颜色/色号、打印样式，避免只按颜色判断线宽。
  • 优先配置高频线宽，例如 0.18、0.25、0.35、0.50 等项目常用级别。
  • 保存配置文件，作为项目样板或试点模板的一部分。
STEP 2

导入 DWG：Insert → Import CAD

导入时确认 Current View Only、颜色、图层、单位和定位。对于二维详图资源，通常建议在目标详图视图或绘图视图内导入，减少 DWG 在非目标视图中扩散。

  • 二维表达建议勾选 Current View Only，让导入内容只服务当前视图。
  • Import Units 不确定时不要长期依赖 Auto-Detect，先用 CAD 标注尺寸复核。
  • 定位方式保持一致，避免不同批次导入产生偏移。
STEP 3

设置对象样式：Manage → Object Styles → Imported Objects

DWG 导入后，Revit 会按导入类别和子类别显示 CAD 图层。把映射表中的 Revit 对象样式、导入的图元样式、线样式/线宽逐项落实到 Imported Objects，形成底层默认显示规则。

  • 按 CAD 图层名称核对 Revit 中出现的导入子类别。
  • 把“线样式/线宽”字段落实到 Revit 线样式和线宽号。
  • 不要在单个视图里随手覆盖底层对象样式，否则后续难以追踪。
STEP 4

控制视图显示：VV/VG → Imported Categories

Autodesk 官方文档说明，Visibility/Graphics 的 Imported Categories 可以控制导入或链接 CAD 文件中各图层的显示。这里适合处理“某张图要不要显示某类 CAD 图层”的视图级控制。

  • 展开 DWG 文件名，逐层核对图层显示状态。
  • 只把视图相关的开关放在 VV/VG；项目通用样式仍应回到 Object Styles。
  • 把成熟组合固化为视图样板，避免每张图重复配置。
STEP 5

处理 Hatch 与填充比例

如果填充样式不需要跟随视口比例变动，可以在 CAD 中炸开成为线段，增加在 Revit 里显示的稳定性；如果需要与视口比例联动，则需在 Revit 中配置填充图案和填充区域的映射关系。

  • 非比例联动填充：CAD 端先炸开 Hatch，再导入 Revit 校核显示。
  • 比例联动填充：在 Revit 中建立对应填充图案，映射到填充区域或资源族表达。
  • 复杂 Hatch 不建议直接作为黑箱导入后固化，先做小样验证。
STEP 6

按需 Explode 并整理为二维资源

Autodesk 官方文档区分 Partial Explode 与 Full Explode，并提示链接文件不能 explode，且导入符号超过一定元素数量时不能 explode。实操中建议只对确实需要编辑、沉淀的部分进行分级处理。

  • 优先尝试 Partial Explode，保留可控层级。
  • Full Explode 只用于需要转成 Revit 线、文字、填充区域的最终资源。
  • 爆炸后立刻整理为组、详图项目族或可复用资源，不保留散乱图元。

03 · Check List

导入后的复核清单

Revit 设置完成后，不以“屏幕看起来差不多”为验收标准，而以映射表、视图样板和打印结果共同确认。

显示复核

  • CAD 图层是否能在 Imported Categories 中找到。
  • 线型、颜色、线宽是否与映射表一致。
  • 是否存在被视图覆盖导致的临时显示差异。

比例复核

  • 1:20、1:50、1:100 等典型比例下线宽是否可读。
  • Hatch 是否需要跟随视口比例变化。
  • 文字高度、箭头、标注线是否满足出图习惯。

资源复核

  • 可复用内容是否已归入详图项目族、组或资源库。
  • 参数化项是否已在映射表中标记可参变。
  • 待定和待确认项是否禁止进入正式模板。
推荐原则

Object Styles 管底层默认，VV/VG 管当前视图，View Template 管批量复用，Mapping Table 管追溯依据。四者分工清楚，后续维护成本才会低。
