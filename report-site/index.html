01 · 项目背景

问题不是“会不会画”，而是如何在 Revit 里高效、稳定、可复用地表达幕墙设计。

本次驻场工作围绕迪士尼项目的 Revit 出图要求展开。幕墙方案提资以 CAD 节点图为主， 但项目成果需要进入 Revit 图纸体系。对于不熟悉 Revit 的设计师来说， 二维节点表达、三维翻模和族资源复用都容易形成效率瓶颈。

我的角色不仅是提出操作建议，而是在参与实际建模和出图的过程中验证路径： 一方面解决二维节点图从 CAD 到 Revit 后的表达保真问题，另一方面承担定制插件开发， 并以 AI 辅助的 Vibe Coding 快速迭代插件原型，让幕墙建模从重复手工操作逐步转向规则驱动。

核心任务

两条工作线同步推进

A

CAD to Revit 二维节点表达

解决小尺寸倒角、曲线、填充、线型和遮罩在 Revit 二维出图中的表达问题。

  • 保留 CAD 已完成的节点绘制成果
  • 规避 Revit 小于 0.8mm 线段表达限制
  • 用 Revit 二维线补齐剩余表达
B

Revit 三维翻模效率提升

通过 Revit 二次开发与 AI Vibe Coding，把重复建模步骤沉淀为路径、规则、模板和资源驱动的插件能力。

  • 降低重复分格、面板替换和龙骨布置工作量
  • 复用项目资源与本地素材库
  • 保留原生 Revit 模型可编辑性

02 · 二维节点表达

CAD 图纸不是问题本身，关键是找到进入 Revit 后仍能保留表达质量的转换方法。

CAD 节点图
炸开含填充线段
导入 Revit
二维线补充
出图表达

已验证的解决路径

对已在 CAD 中绘制完成的节点图，先在 CAD 里将带填充的线段炸开，再导入 Revit， 可以较稳定地保持填充表达，约保留 80% 的 CAD 样式。

剩余约 20% 的差异，例如线型、遮罩、局部二维表达控制， 通过 Revit 内置二维线、遮罩区域和图纸视图补充完成。这样既规避 Revit 对极小线段的表达限制， 也避免把已经完成的 CAD 节点全部重画。

03 · 三维建模提效

Revit-CW-plugin 的核心思路：选择路径或面，套用规则模板，生成可继续编辑的 Revit 幕墙模型。

建模痛点

重复性工作高，分格、面板替换、龙骨布置依赖人工逐项处理，方案调整后返工明显。

资源痛点

已有 Rhino 模型进入 Revit 后难以作为可编辑幕墙继续深化，族资源和规则难以复用。

插件路线

基于 C#、Revit API、WPF 参数面板和 JSON 模板，并引入 AI Vibe Coding 进行快速开发与验证，建立幕墙系统、分格和面板替换规则。

AI 开发亮点

以实际业务痛点为提示词输入，将需求拆解、代码生成、调试修正和文档沉淀串成连续迭代流程，缩短插件从想法到可测 MVP 的周期。

1选择路径 / 墙线 / 面
2选择幕墙系统模板
3输入标高与分格规则
4调用项目资源与素材库
5生成原生 Curtain Wall
6输出摘要与异常提示
